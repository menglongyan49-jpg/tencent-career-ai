# 岗位能力要求知识库

## 技术类岗位

### 后端开发工程师

**核心能力要求**：
1. **编程语言**：精通 Java/Go/C++ 至少一门
2. **数据结构与算法**：扎实的基础，能解决复杂问题
3. **系统设计**：分布式系统、高并发、高可用
4. **数据库**：MySQL、Redis、消息队列
5. **工程能力**：代码质量、测试、CI/CD

**面试高频考点**：
- LeetCode 中等难度题目
- 系统设计：秒杀系统、短链接、消息队列
- MySQL 索引优化、事务隔离级别
- Redis 数据结构、持久化、分布式锁
- 微服务架构、服务治理

**成长建议**：
- 刷题：LeetCode 200+ 题
- 项目：完成一个完整的后端项目
- 开源：参与开源项目贡献
- 博客：写技术博客记录成长

## 简历能力呈现建议

### 项目经历写法 (STAR法则)

```
Situation (情境)：项目背景是什么？
Task (任务)：你的任务是什么？
Action (行动)：你做了什么？
Result (结果)：取得了什么成果？(量化)
```

### 示例

**不好的写法**：
> 负责后端开发，使用了Spring Boot和MySQL。

**好的写法**：
> 负责电商系统后端开发(S)，面临高并发秒杀场景(T)，
> 设计并实现了基于Redis的分布式锁和消息队列削峰方案(A)，
> 成功支撑了10万QPS的秒杀活动，系统稳定性99.99%(R)。
