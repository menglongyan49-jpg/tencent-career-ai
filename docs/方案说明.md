# 「未来鹅」方案说明

## 一、问题诊断

### 1.1 用户痛点

在与高校学生的接触中，我们发现大学生在不同阶段都面临"未来去哪、做什么、怎么准备"的持续困惑：

| 阶段 | 核心困惑 | 痛点本质 |
|-----|---------|---------|
| 大一 | 对互联网行业和腾讯充满好奇，但缺少系统认知入口 | 认知空白 |
| 大二 | 思考专业与职业关联，但缺乏经验 | 方向迷茫 |
| 大三 | 进入实习探索期，想了解岗位/文化/培养体系 | 信息差 |
| 大四/研究生 | 正式求职窗口，面临焦虑和信息差 | 决策压力 |

### 1.2 现有方案缺陷

目前企业的校招信息触达方式相对单向（公众号推文、宣讲会、直播、官网、课程等），存在以下问题：

1. **时间断层**：主要面向"已有明确求职意向"的高年级学生，中低年级存在服务真空
2. **情感缺失**：单向传播缺少"一对一陪伴"的情感连接
3. **个性化不足**：通用化内容无法满足不同专业、不同阶段学生的差异化需求

---

## 二、方案设计

### 2.1 核心理念

> **"真正的陪伴不是单向的千人一面，而是双向选择的契合。"**

将企业招聘前置为「成长陪伴」，通过持续的价值输出建立信任，让学生在成长中自然形成对企业的认同。

### 2.2 产品定位

**「未来鹅」** —— 原生于 AI 的在校大学生求职成长陪伴智能体，采用多智能体协同架构与长期记忆机制，打造具有千人千面性格的"拟人化"导师天团。

### 2.3 核心创新

#### 创新一：性格"捏脸"系统
打破冰冷的客服感，学生可自定义陪伴体的沟通风格：
- 🎀 **知心学姐**：温柔细腻，善解人意
- 🤓 **硬核极客**：技术导向，直击痛点
- 👔 **严厉面试官**：模拟真实面试场景
- 🌟 **全能导师**：平衡专业与温度

系统将动态重写底层 Prompt，实现定制化的人格陪伴。

#### 创新二：多角色协同诊断
复杂问题不依赖单一模型幻觉，系统底层的多个 Agent 相互协同：
- **情绪安抚** → **能力诊断** → **路径规划** → **行动建议**

#### 创新三：长期记忆沉淀
陪伴体能记住学生的年级、技术栈变化、情绪起伏，让每一次对话都基于历史上下文"继续"，而非"重启"。

## 三、AI 工具选型理由

### 3.1 技术栈选型

| 层级 | 技术选型 | 选型理由 |
|-----|---------|---------|
| 前端交互 | **Streamlit** | 极简构建、原生流式支持、快速迭代 |
| Agent 引擎 | **LangChain + LangGraph** | 成熟的多Agent框架、状态管理完善 |
| 大模型 | **Claude API** | 强大推理、长上下文、流式输出 |
| 向量数据库 | **ChromaDB** | 轻量级、本地部署、快速检索 |
| 对话存储 | **SQLite** | 轻量级、无需额外服务 |

### 3.2 多智能体架构

```
用户输入 → Memory Router (路由管家)
              ↓
    ┌─────────┼─────────┬─────────┐
    ↓         ↓         ↓         ↓
Career    Emotional  Interview  Tencent
Navigator Supporter  Coach      Expert
(战术导师) (情绪树洞) (面试教练) (鹅厂专家)
```

## 四、关键配置

### 4.1 Agent 角色配置

| Agent | 职责 | 触发条件 |
|-------|------|---------|
| Memory Router | 记忆管理、意图路由 | 所有输入 |
| Career Navigator | 求职路径、能力对标 | 技术问题、职业规划 |
| Emotional Supporter | 情绪疏导、心理支持 | 焦虑词汇检测 |
| Interview Coach | 模拟面试、简历优化 | 面试/简历关键词 |
| Tencent Expert | 鹅厂文化、岗位解读 | 腾讯相关问题 |

### 4.2 情绪检测机制

系统内置焦虑关键词库，当检测到高频焦虑词汇时，优先路由给情绪树洞进行疏导。

### 4.3 长期记忆机制

- 用户档案：年级、专业、目标、兴趣
- 技能追踪：记录用户提到的技术栈
- 情绪历史：追踪情绪变化趋势
- 成长里程碑：记录关键成长节点
